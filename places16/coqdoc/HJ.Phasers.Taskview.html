Library HJ.Phasers.Taskview


Section Defs.
The taskview represents the local view of a phaser that a registered task has. It records: the number of signals the task issued, the number of waits the task issued, and the task's registration mode. The taskview is changed by invoking signal and wait on a phaser (or next).

  Record taskview := TV {
    signal_phase: nat;
    wait_phase: nat;
    mode: regmode
  }.

Creation

When creating a phaser, the creator is the only task registered in that phaser. Value make represents the first taskview that is initialized in SIGNAL_WAIT mode and accounts for zero signals and zero waits.

  Definition make := TV 0 0 SIGNAL_WAIT.

Basic mutators

We define the mutators of a taskview. These are only used internally.

  Definition set_signal_phase v n := TV n (wait_phase v) (mode v).
  Definition set_wait_phase v n := TV (signal_phase v) n (mode v).
  Definition set_mode v m := TV (signal_phase v) (wait_phase v) m.

Signal

The two mutators available of taskviews are signal and wait, defined following. The semantics of signal depends on the registration mode of the issuer (this is captured by SignalPre: SIGNAL_WAIT tasks interleave signals and waits and the first operation is a signal; SIGNAL_ONLY tasks can always issue a signal.

  Inductive SignalPre v: Prop :=
    | signal_pre_w:
      mode v = SIGNAL_WAIT ->
      wait_phase v = signal_phase v ->
      SignalPre v
    | signal_pre_s:
      mode v = SIGNAL_ONLY ->
      SignalPre v.

The operation simply increments the signal phase.

  Definition signal v := set_signal_phase v (S (signal_phase v)).

At the language-level, however, invoking signal multiple times is not considered an error, yet only the first signal after a wait is observed. Precondition SignalPre v models the first signal after a wait. Function try_signal can be invoked without a pre-condition and models the user-facing signal operation.

  Definition try_signal v :=
  match mode v with
  | SIGNAL_WAIT => set_signal_phase v (S (wait_phase v))
  | _ => signal v
  end.

Wait

The wait operation increments the wait-phase. A SIGNAL_WAIT task must have signalled before invoking wait. Tasks registered as WAIT_ONLY are always capable of invoking a wait. SIGNAL_ONLY tasks cannot issue a wait.

  Inductive WaitPre v: Prop :=
    | wait_pre_wo:
      mode v = WAIT_ONLY ->
      WaitPre v
    | wait_pre_sw:
      mode v = SIGNAL_WAIT ->
      S (wait_phase v) = signal_phase v ->
      WaitPre v.

  Definition wait v := set_wait_phase v (S (wait_phase v)).

Small-step operational semantics

The operational semantics not only defines a closed set of operations that can operate on a value (here a taskview), but also defines the preconditions of each operation. Contrary to eval, relation Reduces defines a precondition to wait, called WaitPre: in order to issue a wait, the wait phase must be behind (smaller than) the signal phase. We define CanWait so it can be reused in other developments.
The operational semantics of taskviews defines a closed set of operations op that can change taskviews, signal and wait.

  Inductive op := SIGNAL | WAIT.

Function eval interprets the given operation o as the appropriate function.

  Definition eval o :=
  match o with
  | SIGNAL => signal
  | WAIT => wait
  end.

  Inductive Reduces v : op -> taskview -> Prop :=
    | tv_reduces_signal:
      SignalPre v ->
      Reduces v SIGNAL (signal v)
    | tv_reduces_wait:
      WaitPre v ->
      Reduces v WAIT (wait v).

End Defs.

Section Facts.

We can simplify an expression that sets the mode it already has. This lemma also shows the correctness of set_mode.

  Lemma set_mode_ident:
    forall v,
    set_mode v (mode v) = v.

  Lemma mode_set_mode_rw:
    forall v r,
    mode (set_mode v r) = r.

A property of correctness: set_mode preserves the signal phase.

  Lemma set_mode_preserves_signal_phase:
    forall v r,
    signal_phase (set_mode v r) = signal_phase v.

A property of correctness: set_mode preserves the wait phase.

  Lemma set_mode_preserves_wait_phase:
    forall v r,
    wait_phase (set_mode v r) = wait_phase v.

The initial taskview is registered in SIGNAL_WAIT mode.

  Lemma make_mode:
    mode make = SIGNAL_WAIT.

The initial taskview accounts for zero signals.

  Lemma make_signal_phase:
    signal_phase make = 0.

The initial taskview accounts for zero waits.

  Lemma make_wait_phase:
    wait_phase make = 0.


  Lemma signal_preserves_mode:
    forall v,
    mode (signal v) = mode v.

  Lemma try_signal_preserves_mode:
    forall v,
    mode (try_signal v) = mode v.

  Lemma signal_preserves_wait_phase:
    forall v,
    wait_phase (signal v) = wait_phase v.

  Lemma try_signal_preserves_wait_phase:
    forall v,
    wait_phase (try_signal v) = wait_phase v.

  Lemma try_signal_signal_phase_sw:
    forall v,
    mode v = SIGNAL_WAIT ->
    S (wait_phase v) = signal_phase (try_signal v).

  Lemma signal_phase_set_signal_phase:
    forall v n,
    signal_phase (set_signal_phase v n) = n.

  Lemma signal_so_signal_phase:
    forall v,
    mode v = SIGNAL_ONLY ->
    signal_phase (signal v) = S (signal_phase v).

  Lemma wait_preserves_mode:
    forall v,
    mode (wait v) = mode v.

  Lemma wait_preserves_signal_phase:
    forall v,
    signal_phase (wait v) = signal_phase v.

  Lemma wait_phase_set_wait_phase:
    forall v n,
    wait_phase (set_wait_phase v n) = n.

  Lemma wait_wait_phase:
    forall v,
    wait_phase (wait v) = S (wait_phase v).

  Lemma signal_wait_cap_signal_phase:
    forall v,
    SignalPre v ->
    mode v = SIGNAL_WAIT ->
    signal_phase (signal v) = S (wait_phase v).

End Facts.

Section Semantics.

A trivial property we choose to highlight is that Reduces performs an eval on the right-hand side.

  Lemma reduces_spec:
    forall v o v´,
    Reduces v o v´ ->
    v´ = eval o v.

  Lemma reduces_rw_signal:
    forall v v´,
    Reduces v SIGNAL v´ ->
    v´ = signal v.

  Lemma reduces_rw_wait:
    forall v v´,
    Reduces v WAIT v´ ->
    v´ = wait v.

  Lemma eval_preserves_mode:
    forall v o,
    mode (eval o v) = mode v.

  Lemma reduces_preserves_mode:
    forall v o v´,
    Reduces v o v´ ->
    mode v´ = mode v.

  Lemma reduces_signal_inv_sw:
    forall x y z o,
    Reduces x SIGNAL y ->
    Reduces y o z ->
    mode x = SIGNAL_WAIT ->
    o = WAIT.

  Lemma reduces_wait_inv_sw:
    forall x y z o,
    Reduces x WAIT y ->
    Reduces y o z ->
    mode x = SIGNAL_WAIT ->
    o = SIGNAL.

End Semantics.
